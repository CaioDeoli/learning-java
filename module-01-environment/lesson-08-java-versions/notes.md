# Conhecendo as versões do Java
- Como funciona o versionamento do Java

## Linha do tempo das versões do Java
...
- Java 6
  - 2006 -> 2018
- Java 7
  - 2011 -> 2022
- **Java 8**
  - 2014

- Até aqui nao tinha nenhuma previsibilidade de lançamento de versão
- De 6 em 6 meses lança uma release nova
  - Geralmente em março e setembro de cada ano
- Feature release deixa de ter suporte a cada 6 meses
- Se esta usando feature release, sempre atualize para a mais recente para ter o suporte
- LTS sendo lançadas de 3 em 3 anos

- Java 9 (2017.2) - Feature release (FR): versao para demonstracao para depois lançar a definitiva
- Java 10 (2018.1) - FR
- **Java 11** (2018.2) - LTS -> long term support. atualizacoes até 2027
- Java 12 (2019.1) - FR
- Java 13 (2019.2) - FR
- Java 14 (2020.1) - FR
- Java 15 (2020.2) - FR
- Java 16 (2021.1) - FR
- **Java 17** (2021.2) - LTS

- Mudança de LTS para 2 em 2 anos

- Java 18 (2022.1) - FR
- Java 19 (2022.2)
- Java 20 (2023.1)
- **Java 21** (2023.2)
- Java 22 (2024.1)
- Java 23 (2024.2)
- Java 24 (2025.1)
- **Java 25** (2025.2)

- Java 8 ainda usado em projetos legados
- Java 11 é menos crítico
- Projetos novos use o LTS mais recente caso possível
- Evite usar a FR para iniciantes e projetos grandes
- Mas é legal testar as FR para voce ficar por dentro das funcionalidades que virão
- Tem empresas que mantem essas versões legado LTS de maneira paga
- Usaremos o Java 17 nesse curso mas voce pode usar a LTS que tiver mais recente porque dificilmente algo é removido da linguagem
